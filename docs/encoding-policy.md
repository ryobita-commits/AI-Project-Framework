# エンコーディング方針

## 目的

日本語ファイルと英語ファイルが混在しても文字化けしないように、repo 内の文字コードと改行コードの運用を統一する。

## 基本ルール

- テキスト系ファイルは `UTF-8` を使う
- 可能な限り `BOM なし UTF-8` を使う
- Markdown、YAML、JSON、コード、設定ファイルも同じく `UTF-8` を使う
- 日本語を含むファイルでも `Shift_JIS` や `CP932` を使わない

## 対象ファイル

- `*.md`
- `*.txt`
- `*.yml`
- `*.yaml`
- `*.json`
- `*.ts`, `*.tsx`, `*.js`, `*.jsx`
- `*.py`
- `*.ps1`
- その他のテキストベースの設定ファイル

## 改行コード

- repo 上の標準は `LF` を推奨する
- Windows 作業環境で一時的に `CRLF` になっても、内容と文字コードの整合を優先する
- 大量の改行コード差分だけを発生させる編集は避ける

## 編集時の注意

- エディタ保存時は `UTF-8` を選ぶ
- 文字化けして見える場合は、まず読み込み側の encoding を確認する
- PowerShell で確認するときは `Get-Content -Encoding utf8` を優先する
- 新規ファイル作成時は、既存の日本語 Markdown と同じ運用に合わせる

## レビュー時の確認項目

- 日本語が文字化けしていないか
- 英数字や記号が意図せず壊れていないか
- encoding 変更だけの不要差分が混ざっていないか
- 改行コードだけのノイズ差分が増えていないか
