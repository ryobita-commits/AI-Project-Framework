# Feature Spec Sample

## Feature Name

Combat Core Loop

## Purpose

Define the smallest playable loop that proves the core experience.

## User Or Player Value

The user gets a clear, repeatable interaction that demonstrates the main promise of the product.

## Scope

- Included: movement, attack, enemy response, win or fail condition
- Excluded: long-term progression, multiplayer, narrative systems

## Acceptance Criteria

- A session can be started and finished
- Core interaction is understandable without external explanation
- Main loop can be repeated without blocking issues

## Dependencies

- Project type confirmed
- Engine or runtime confirmed
- UX flow drafted

## Risks

- Scope may still be too broad
- Technical assumptions may change

## Implementation Notes

- Keep the first version thin and testable
