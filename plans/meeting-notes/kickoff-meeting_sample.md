# Meeting Note Sample

## Date

2026-03-26

## Topic

Project kickoff and scope framing

## Participants

- moderator: pm_moderator
- advisors: pm_analyst, pm_advocate, pm_critic
- humans: project owner

## Decision Question

What should the first milestone focus on so the team can start execution with low ambiguity?

## Shared Goal

Choose a narrow first milestone that reduces risk and creates a usable planning baseline.

## Options Considered

- Create a combat-focused vertical slice
- Create a broader prototype with multiple mechanics
- Spend one more cycle on discovery before committing

## Analyst Position

The vertical slice option offers the clearest planning surface. It limits scope, allows measurable quality gates, and makes architecture decisions easier.

## Advocate Position

The team should move into a concrete vertical slice quickly. Shipping one playable outcome creates momentum and forces useful prioritization.

## Critic Position

A vertical slice still fails if the core loop is not explicitly defined. Engine choice and content ambition must be constrained before implementation begins.

## Human Input

The first milestone should stay small. The owner prefers a playable result over extensive pre-production.

## Decision

Proceed with a narrowly scoped first milestone and define one concrete feature spec next.

## Rationale

This gives the team enough structure to start design and implementation while keeping rework manageable.

## Risks

- Engine choice is still open
- MVP may drift without a written feature spec
- Test scope is not defined yet
