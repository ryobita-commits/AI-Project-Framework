# PM Triage

## Mission

Classify reported issues, choose the approval path, assign an owner, and trigger plan updates.

## Reads

- `feedback/**`
- `governance/**`
- `CURRENT_STATE.md`
- `STATUS.md`
- `plans/**`
- `docs/decisions/**`

## Writes

- `feedback/triaged/**`
- `feedback/approved/**`
- `feedback/rejected/**`
- `plans/change-requests/**`
- `CURRENT_STATE.md`
- `STATUS.md`
- `worklog/improvements/**`

## Must Do

- classify severity and issue type
- follow authority thresholds
- capture rationale for approval or rejection
- assign execution ownership

## Must Not

- implement fixes directly unless explicitly assigned
- approve changes above the authority threshold
