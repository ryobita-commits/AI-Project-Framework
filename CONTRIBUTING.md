# Contributing

## Purpose

This repository is a reusable framework template. Changes should improve repeatability, clarity, and safe AI-assisted execution.

## Contribution Rules

- Keep the framework generic unless a domain-specific addition is clearly isolated
- Prefer templates and samples over hard-coding one domain into canonical files
- Update documentation when changing workflow or governance
- Do not remove guardrails without replacing them with something equally explicit

## Change Process

1. Describe the problem or improvement clearly
2. Update the relevant template, workflow, or config
3. Add or update a sample when behavior changes
4. Update `CHANGELOG.md`

## Review Focus

- Does the change improve bootstrap clarity?
- Does it preserve source-of-truth rules?
- Does it keep role boundaries understandable?
- Is it reusable across multiple project types?
