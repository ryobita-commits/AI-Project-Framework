# PROJECT SAMPLE

## Project Type

- Type: game

## Working Title

- Title: Project Ember Loop

## One-Line Pitch

- Pitch: This project will create a replayable tactical action game for PC players so they can enjoy short, mastery-driven combat runs.

## Goal

- Primary goal: Deliver a small but polished combat-focused demo.
- Problem being solved: The team needs a tightly scoped first release target.
- Why now: The project is in zero-to-one discovery and needs a concrete direction.

## Target Users Or Stakeholders

- Primary: PC players who enjoy short action sessions
- Secondary: Streamers and early playtesters
- Internal stakeholders: PM, design, engineering, QA

## Output Or Product Shape

- Main deliverable: Playable PC demo
- Supporting deliverables: Design docs, test plan, release checklist
- Expected usage context: Downloadable alpha build

## MVP Or First Milestone

- Must have: One combat loop, one enemy family, one arena, one progression hook
- Should have: Basic tutorial and settings menu
- Deferred for later: Meta progression, online ranking, controller remapping

## Non-Goals

- Open world content
- Online multiplayer
- Live service operations

## Success Metrics

- Outcome metric: Playtesters finish 3 runs in one session
- Delivery metric: Vertical slice shipped by milestone date
- Quality metric: No P0 or P1 issues in demo build

## Constraints

- Team size: 1-3 people plus AI agents
- Budget: low
- Schedule: 8 weeks to demo
- Technology constraints: Unity or Godot only
- Compliance or external constraints: none yet

## Technical Assumptions

- Platform: PC
- Runtime or engine: Unity
- Backend required: no
- Integrations: analytics later if needed

## Open Questions

- Unity or Godot final choice
- How much narrative is needed in the first demo

## Bootstrap Status

- Initial questionnaire completed: yes
- Project type confirmed: yes
- First feature or work package defined: no
- First planning meeting held: no
